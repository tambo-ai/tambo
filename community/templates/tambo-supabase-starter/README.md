## Getting Started

### 1. Installation

Clone the repository and install dependencies:

```bash
npm install

### 2. Database Setup
Go to your Supabase SQL Editor and run this command to create the tasks table:

SQL
create table todos (
  id bigint generated by default as identity primary key,
  text text not null,
  is_complete boolean default false,
  created_at timestamp with time zone default timezone('utc'::text, now()) not null
);

-- Insert a sample task
insert into todos (text, is_complete) values ('Star this repo!', false);
Important: Ensure Row Level Security (RLS) is disabled for this table, or configure policies to allow anonymous access for this demo.

### 3. Environment Variables
Rename example.env.local to .env.local and add your keys:

Code snippet
VITE_TAMBO_API_KEY=your_tambo_api_key
VITE_SUPABASE_URL=your_supabase_project_url
VITE_SUPABASE_ANON_KEY=your_supabase_anon_key

### 4. Run the App
Bash
npm run dev
Open http://localhost:5173 in your browser.

How to Use
View Tasks: Type "Show my tasks" or "What do I need to do?"

Filter: Type "Show only pending tasks" or "Show completed items."

Interact: Click the checkboxes in the UI to toggle task status. The AI will confirm the update.

Project Structure
src/components/tambo/TaskList.tsx: The component that renders the list and handles Supabase logic.

src/App.tsx: The main chat interface with the Tambo Provider configuration.

src/lib/supabase.ts: The database connection client.

Built for the Tambo Templates Challenge.
```
