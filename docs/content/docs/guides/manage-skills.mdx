This guide walks you through creating and managing skills for your Tambo project. You can manage skills through the Tambo dashboard or the CLI.

## Prerequisites

- A Tambo project using OpenAI or Anthropic as the LLM provider
- A model that supports skills (see [Skills concept page](/concepts/skills#model-support) for the full list)

If your project uses an unsupported provider or model, the skills section in your project settings shows a warning with instructions to switch.

## Create a Skill in the Dashboard

1. Open your project in the Tambo dashboard
2. Navigate to **Settings**
3. Scroll to the **Skills** section
4. Click **Create Skill**
5. Fill in the name (kebab-case), description, and instructions
6. Click **Save**

The skill is automatically uploaded to your LLM provider and enabled for all new conversations.

## Import a Skill File

You can import skills from markdown files with YAML frontmatter.

1. In the Skills section, click **Import** or drag and drop a `.md` file
2. Tambo parses the frontmatter for the name and description
3. Review and confirm the imported content
4. Click **Save**

### Skill file example

```markdown
---
name: code-reviewer
description: "Review code for bugs, security issues, and best practices"
---

You are a code review assistant. When the user shares code:

1. Check for bugs and logic errors
2. Identify security vulnerabilities
3. Suggest improvements for readability and performance
4. Provide specific line-by-line feedback
5. Explain your reasoning for each suggestion
```

## Manage Skills via CLI

The Tambo CLI provides full skill management from the terminal. All commands require authentication (`npx tambo auth login`) and a project API key in `.env.local`.

### List skills

```bash
npx tambo skills list
```

### Add a skill from a file

```bash
npx tambo skills add code-reviewer.md
```

### Export a skill to edit locally

```bash
npx tambo skills get code-reviewer > code-reviewer.md
# edit the file...
npx tambo skills update code-reviewer.md
```

### Delete a skill

```bash
npx tambo skills delete code-reviewer
```

For the full CLI reference, see [CLI Skills Commands](/reference/cli/commands/skills).

## Enable and Disable Skills

Toggle individual skills on or off from the Skills section in your project settings. Disabled skills remain stored but are not passed to the LLM at inference time.

This is useful for A/B testing skill configurations or temporarily disabling a skill without deleting it.
