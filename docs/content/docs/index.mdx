Tambo is an open-source toolkit for building AI-powered React apps. Register your components, and the agent decides which one to render based on user messages.

"Show me sales by region" renders your `<Chart>`. "Add a task" updates your `<TaskBoard>`. The agent picks the component and streams the props.

## What you can build

From building completely new agent-powered experiences to collapsing multi-step screen navigation in complex enterprise apps. You write React components; Tambo handles model calls, streaming, tool execution, and message history.

### New agent-powered experiences

Register your components and let the agent figure out which ones to render:

- **Recipe app** → register `<MealPlan>`, `<RecipeCard>`, `<ShoppingList>`, `<NutritionSummary>`. A user says "plan my meals for the week using what's in my fridge" and the agent renders a meal plan with nutrition totals and a shopping list for missing ingredients.
- **Fitness tracker** → register `<WorkoutPlan>`, `<ProgressChart>`, `<ExerciseDemo>`. A user says "build me a 4-week strength program for someone who can work out 3 days a week" and the agent generates a structured plan across your components.
- **Personal finance** → register `<BudgetBreakdown>`, `<SpendingChart>`, `<SavingsGoal>`. A user says "show me where my money went last month and suggest where to cut" and the agent renders analytics with recommendations.

Building this from scratch means wiring up model integration, streaming, state management, and tool execution yourself. Tambo gives you that layer out of the box.

### Workflow compression for existing apps

For apps with existing UI, agent interfaces compress multi-step workflows into single interactions:

- **"Brain dump my last meeting with Sarah at Acme"** → creates the contact, schedules the follow-up, adds the note, and updates the CRM. Tasks that normally span 4 screens and multiple forms.
- **"Mark all leads older than 90 days as inactive"** → queries, shows a preview UI for review, and applies on approval.
- **"Pull up customers with no activity in 90 days, give me a UI to review and archive"** → runs the query and renders an interactive results component inline.

Most relevant in apps with deep navigation or complex forms.

### Try it with a coding agent

The fastest way to evaluate Tambo is to build something. Install the [Tambo skill](/guides/coding-agent-skills) and ask your coding agent to create a demo:

```bash
npx skills add tambo-ai/tambo
```

Then prompt: "Create a Tambo app with a weather dashboard." You'll have a working generative UI in under 5 minutes.

For existing codebases, the agent scans your code, identifies components, generates schemas (Zod, ArkType, Valibot, or any Standard Schema-compatible library), and runs tests. The human role is reviewing and approving the agent's work. Most teams have a working POC in an afternoon. Starting with 5–10 high-value components is a good way to validate the pattern before scaling to a full integration.

## How Tambo works

Two component patterns, one toolkit.

### Generative components

Rendered once in response to a message. Use these for charts, data visualizations, summary cards—anything that displays a result.

## Core workflow

### 1. Register your components

Tell the agent which components it can use:

```tsx
import { TamboProvider } from "@tambo-ai/react";

export function Home() {
  return (
    <TamboProvider
      components={myTamboComponents}
      tools={myTamboTools}
      apiKey={tamboApiKey}
    >
      <MyAiApp />
    </TamboProvider>
  );
}
```

For apps with signed-in users, pass a `userToken` to enable per-user auth. See [User Authentication](/concepts/user-authentication) for details.

### 2. Use Tambo hooks

Send messages and render responses:

```tsx
const { messages } = useTambo();
const { value, setValue, submit, isPending } = useTamboThreadInput();

// Render messages with components
{
  messages.map((message) => (
    <div key={message.id}>
      {Array.isArray(message.content) ? (
        message.content.map((part, i) =>
          part.type === "text" ? <p key={i}>{part.text}</p> : null,
        )
      ) : (
        <p>{String(message.content)}</p>
      )}
      {message.renderedComponent}
    </div>
  ));
}
```

### 3. Add MCP integrations

Connect pre-built integrations or your own MCP servers:

```tsx
import { MCPTransport } from "@tambo-ai/react/mcp";

const mcpServers = [
  {
    name: "filesystem",
    url: "http://localhost:8261/mcp",
    transport: MCPTransport.HTTP,
  },
];

<TamboProvider components={components} mcpServers={mcpServers}>
  <App />
</TamboProvider>;
```
